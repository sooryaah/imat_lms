### 3. Real-Time Chat Implementation (WebSocket)

**Technology Stack:**
- **Framework**: Django Channels
- **Protocol**: WebSocket (ws:// or wss://)
- **Message Queue**: Redis (production) or In-Memory (development)
- **Authentication**: JWT token passed during WebSocket handshake

**WebSocket Endpoints:**

```
ws://localhost:8000/ws/community/chat/{community_id}/
ws://localhost:8000/ws/notifications/
```

**Message Format (JSON):**

```json

  "type": "chat_message",
  "message": "Hello everyone!",
  "message_type": "text",
  "community_id": 1
}

// Server â†’ All clients (broadcast)
{
  "type": "chat_message",
  "message_id": 42,
  "message": "Hello everyone!",
  "message_type": "text",
  "sender_id": 5,
  "sender_email": "student@example.com",
  "created_at": "2025-12-15T10:30:00Z"
}

// Typing indicator
{
  "type": "typing",
  "is_typing": true,
  "community_id": 1
}

// Message read receipt
{
  "type": "read_receipt",
  "message_id": 42
}
```

**Real-Time Features:**

1. **Instant Message Delivery**
   - Message saved to database with status="sent"
   - Broadcast to all WebSocket connections in the group
   - Status changes: sent â†’ delivered â†’ read

2. **Typing Indicators**
   - Client sends "user is typing" event
   - Broadcast to all other users in the group
   - Automatically clear after 3 seconds of inactivity

3. **Read Receipts**
   - Track who has read messages (ChatMessageReadReceipt)
   - Show "read by X people" indicator
   - Message status updated to "read"

4. **User Presence**
   - Track online users in each community
   - "User joined" event when connecting
   - "User left" event when disconnecting

5. **Message Editing & Deletion**
   - Soft delete (mark as deleted, don't remove from DB)
   - Edit timestamps tracked (is_edited, edited_at)
   - All changes synced via WebSocket

### 4. Notification System

**Types of Notifications:**

| Type | Trigger | Recipients |
|------|---------|-----------|
| `new_post` | New discussion post created | Community members |
| `new_reply` | Reply to user's post | Post author |
| `new_mention` | User mentioned in post | Mentioned user |
| `new_message` | Direct message received | Recipient |
| `post_approved` | Moderator approved post | Post author |
| `post_rejected` | Moderator rejected post | Post author |
| `moderator_action` | Post flagged/moderated | Community moderators |

**Notification Delivery Channels:**

1. **In-App Notifications** (Real-time via WebSocket)
   - Instant delivery to connected users
   - Stored in database for offline viewing
   - Accessible via `GET /api/community/notifications/`

2. **Email Notifications** (Optional, async)
   - Sent via Celery tasks
   - User can control frequency in preferences
   - Includes email_notifications flag in GroupMember

3. **Push Notifications** (Mobile)
   - Firebase Cloud Messaging (FCM) for Flutter
   - push_notifications flag in GroupMember
   - Configured in mobile app

**Creating a Notification (Server-side):**

```python
from community.models import Notification
from channels.layers import get_channel_layer
import asyncio

# Create notification in database
notification = Notification.objects.create(
    recipient=user,
    notification_type='new_post',
    title='New Discussion Post',
    message='John posted in Spoken English program',
    actor=post.author,
    community=community,
    post=post
)

# Send real-time notification via WebSocket
channel_layer = get_channel_layer()
asyncio.run(channel_layer.group_send(
    f'notifications_{user.id}',
    {
        'type': 'send_notification',
        'notification_id': notification.id,
        'notification_type': notification.notification_type,
        'title': notification.title,
        'message': notification.message,
        'actor': notification.actor.full_name,
        'timestamp': notification.created_at.isoformat(),
    }
))

# Send email notification (async)
send_notification_email.delay(notification.id)
```

### 5. API Endpoints Summary

**Community Groups**
```
GET    /api/community/groups/                    # List all communities
POST   /api/community/groups/                    # Create (admin only)
GET    /api/community/groups/{id}/              # Get details
POST   /api/community/groups/{id}/join/         # Join community
POST   /api/community/groups/{id}/leave/        # Leave community
GET    /api/community/groups/{id}/members/      # List members
GET    /api/community/groups/{id}/posts/        # List posts
POST   /api/community/groups/{id}/posts/        # Create post
GET    /api/community/groups/{id}/messages/     # List chat messages
```

**Discussion Posts**
```
GET    /api/community/posts/                    # List posts (paginated)
POST   /api/community/posts/                    # Create post
GET    /api/community/posts/{id}/              # Get post details (increments view count)
PUT    /api/community/posts/{id}/              # Update post (owner only)
DELETE /api/community/posts/{id}/              # Delete post (soft delete)
POST   /api/community/posts/{id}/react/        # Add reaction
POST   /api/community/posts/{id}/approve/      # Approve post (moderator)
POST   /api/community/posts/{id}/reject/       # Reject post (moderator)
```

**Chat Messages (REST)**
```
GET    /api/community/messages/                 # List messages (with community filter)
POST   /api/community/messages/                 # Send message
PUT    /api/community/messages/{id}/           # Edit message (owner only)
DELETE /api/community/messages/{id}/           # Delete message (soft delete)
POST   /api/community/messages/{id}/mark-as-read/  # Mark as read
```

**Notifications**
```
GET    /api/community/notifications/            # List user's notifications
GET    /api/community/notifications/{id}/      # Get notification details
POST   /api/community/notifications/{id}/mark-as-read/   # Mark as read
POST   /api/community/notifications/mark-all-as-read/    # Mark all as read
GET    /api/community/notifications/unread-count/        # Get unread count
```

**WebSocket Connections**
```
ws://localhost:8000/ws/community/chat/{community_id}/
ws://localhost:8000/ws/notifications/
```
